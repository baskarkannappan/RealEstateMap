You are a senior frontend architect and UX engineer. Your task is to significantly improve the UI/UX of the existing website. You are allowed to refactor, redesign, and restructure the UI components, layout, styles, and interaction patterns as needed to achieve a modern, clean, and highly usable interface.

The goal is to create a top-notch user experience that feels modern, fast, intuitive, and mobile-friendly while keeping the application performant and maintainable.

Core UX Goals

The interface must feel clean, modern, and professional.

Navigation should be simple and intuitive.

The UI must work perfectly on desktop, tablet, and mobile devices.

Improve readability, spacing, alignment, and visual hierarchy.

Reduce visual clutter and unnecessary UI elements.

Improve responsiveness and perceived performance.

Layout Improvements

Redesign the page layout to follow a modern responsive design pattern.

Implement a responsive grid system.

Ensure consistent spacing and margins across sections.

Use a content container layout so the content does not stretch too wide on large screens.

Ensure the UI adapts smoothly across different screen sizes.

Avoid horizontal scrolling.

Navigation and Menu System

Improve the navigation to be modern and mobile friendly.

The menu should have:

A compact width on desktop.

Ability to auto-collapse or auto-hide.

Smooth animation when expanding or collapsing.

Implement a hamburger menu that:

Is clearly visible.

Is always accessible.

Appears in the top navigation bar.

Expands the sidebar when clicked.

On mobile:

The menu should be hidden by default.

The hamburger icon should open a slide-in navigation drawer.

The drawer should close when clicking outside or selecting a menu item.

Improve menu usability:

Add icons to menu items.

Highlight the active menu item.

Improve hover and focus states.

Header Improvements

Improve the top header bar.

The header should include:

Hamburger menu

Logo or app name

Search (optional if applicable)

User profile or settings

Notifications (optional)

The header should:

Stay fixed at the top.

Have subtle shadow or separation.

Adjust properly on mobile.

Mobile First Design

The design should follow mobile-first principles.

On small screens:

Stack elements vertically.

Increase touch targets.

Simplify UI components.

Reduce unnecessary content density.

Ensure buttons are easy to tap.

Make sure:

Text is readable.

Components do not overflow.

Layout scales properly.
